Object 1018h: Identity Object


Sub-index 0:

Sub-index 1:

Read: 110, Delta-Q technology vendor identifier.

Sub-index 2:
Scale: NA
Unit: NA
Read: ASCII character codes corresponding to the third through sixth characters of the charger's serial number label. For example, if the charger's serial number label reads DQSB481309100047 then this value will read 0x53423438 ("SB48"). The value is stored as little-endian.

Sub-index 3:
Scale: NA
Unit: NA
Read: The major CANopen revision number in the upper 16-bits; the minor CANopen revision number in the next 8-bits and patch number in last 8bits.

Sub-index 4:
Scale: NA
Unit: NA
Read: Encoded combination of the last 10 characters from the charger's serial number label. For example, if the charger's serial number label is DSQB481309100047 then the last 10 characters are 1309100047. The first two of these characters, 13, represent the year of manufacture. The next two characters are the week of manufacture. And the fifth character, 1,  is the location of manufacture. These are combined as ( 13 * 53 + 9 ) * 16 + 1 = 0x2BA1. The remaining 5 characters are simply encoded as hexadecimal so 000047 = 0x002F. The entire value then is 0x2BA1002F. The value is stored as little-endian.

In the future, Delta-Q serial number labels will be of the format DSB48X1309100047. In this case, sub-index 2 will be based on five serial number characters (i.e., SB48X). Each character will be encoded as 6-bits instead of 8. The encoding is shown below

Object 1029h: Error Behaviour

Scale: NA
Unit: NA
Read/Write: Error class, 0, change to pre-operational NMT state (default); 1, do not change NMT state; 2, change to stopped NMT state; 3-127, reserved; 128-255, manufacturer-specific

When the charger detects a heartbeat timeout or CAN bus-off condition this object determines the charger's CANopen error handling behaviour. For example, if the value of this object is 0 and the charger detects a bus-off or heartbeat timeout it will change its NMT state to pre-operational.

NOTE: The default setting is zero (0), however, the customer may request one of the other settings. Values written to this object are volatile. On restart the charger will recover its configured setting.

See CiA 301 Application Layer and Communication Profile for more information.

Object 2241h: Active Algorithm Request


Sub-index 0:
Scale: NA
Unit: NA
Read: 
0xFFFFFFFF	No active algorithm is set.
Other Values	The actual active algorithm ID on the least significant 16 bits. 			The top 16 bits are reserved
Write: The lower 16-bits store the ID.  The upper 16-bits are a bit field determining the permanence of the selection:

Bit-15 of the upper 16-bit:  algorithm selection permanence
   bit clear: Select algorithm this time only. (it takes effect until a software reset)
   bit set:    Store selection permanently.
Bit-14 of the upper 16-bit: 
   bit clear: The loss of AC will not reset the algorithm back to the permanent algorithm. 
   bit set:    The loss of AC will reset the algorithm back to the permanent algorithm.
Bits 0 to 13 of the upper 16-bit: 
   Reserved bits, all these bits should be set to 0

E.g.  Selecting Algorithm ID 11 permanently:        0x8000000B
         Selecting Algorithm ID 11 non-permanently: 0x0000000B

         Selecting Algorithm ID 28 reset to default algorithm after AC loss: 0x4000001C

If Algorithm ID is not stored in the charger then it will not be set. The current algorithm ID will remain set. If the battery is disconnected or AC is disconnected for a prolonged period then a temporarily set algorithm will revert back to the prior permanent algorithm.
